# SenseTrust Presentation Checklist v2.1

Antes da demo:
- ambiente local;
- branch correta;
- sem dados reais;
- rotas funcionando;
- disclosures visiveis;
- roteiro aberto;
- publico definido.

Durante a demo:
- reforcar prototipo;
- reforcar metadata_only;
- reforcar nao certificacao diagnostica;
- nao improvisar validade juridica;
- nao prometer integracao inexistente.

Depois da demo:
- registrar feedback;
- classificar interesse;
- classificar risco;
- definir proximo passo;
- enviar material autorizado.
